### 결제 페이지 — 흐름을 설계한다는 것

결제 페이지는 입력 흐름이 끊기지 않도록 위에서 아래로 자연스럽게 이어지게 구성했다. 상단에 Progress Indicator를 넣어 현재 위치를 보여주었고, 카드 UI를 상단에 배치해서 지금 어떤 정보를 입력하고 있는지 바로 인식할 수 있도록 했다. 카드 번호, 유효기간, CVC를 실제 입력 순서와 유사하게 배치한 것도 같은 이유였다.

### 계산기 — 기능이 아니라 느낌을 바꿨다

계산기는 이미 익숙한 UI라서 오히려 차별화하기 어려운 영역이었다. 그래서 기능을 추가하기보다 사용할 때의 느낌을 바꾸는 쪽으로 접근했다. 버튼 형태와 컬러 포인트, 개성 있는 폰트로 조금 더 가볍고 유쾌한 느낌을 만들었고, 클릭 시 반응이 보이도록 인터랙션을 넣었다.

`=` 버튼을 실수로 누르는 경우를 고려해서 슬라이드 시 연산자 전환이 가능하도록 설계한 것이 개인적으로 마음에 들었다. 그런데 피드백에서 전체적으로 폰트가 두꺼워서 정보의 위계가 잘 드러나지 않는다는 이야기를 들었다. 개성을 주려다가 가독성을 잃은 셈이었다. 강조와 가독성의 균형이 중요하다는 걸 느꼈다.

---

### Testimonial — 피드백 전후가 가장 달랐다

리뷰 UI를 만들면서 review와 testimonial의 차이를 처음으로 정리해 보았다. review는 비교적 객관적인 평가에 가깝고 점수나 기능 중심이라면, testimonial은 실제 사용자의 경험과 감정을 기반으로 한 신뢰 요소에 더 가깝다. 특히 testimonial은 서비스 제공자가 선택해서 보여주는 경우가 많아서 설득과 공감에 더 가까운 역할을 한다.

그래서 단순한 리뷰 리스트 대신 제품을 중심에 두고 리뷰가 주변에 떠 있는 구조로 구성했다. 사용자 이미지, 별점, 한 줄 후기의 최소 구성만 가져갔는데, 단순해서 오히려 더 직관적으로 보였다.

피드백을 반영해 수정했다. 카드에 유리 효과를 적용하니 배경과 자연스럽게 어우러지면서도 레이어가 분리되어 보였다. 기존보다 시각적인 깊이가 생기면서 후기 요소가 더 부드럽게 강조되는 느낌이 들었다.

### 세 개를 마치고

UI는 예쁘게 만드는 게 아니라 이유를 가지고 배치해야 한다는 것을 이번 과제에서 가장 많이 느꼈다. 결제 화면은 정보 입력 흐름을, 계산기는 실수 방지와 인터랙션을, 후기 UI는 신뢰 전달 방식을 설계하는 것이었다.

그리고 작은 스타일 변화가 전체 인상에 꽤 큰 영향을 준다는 것도 glassmorphism 수정에서 직접 느꼈다.
