DAY 18 실습 과제는 앱 하나를 골라서 Variable Mode로 다크모드를 만드는 것이었다. 다른 앱을 클론하는 대신 지금 직접 기획하고 있는 Boo!를 선택했다. 커플 실시간 채팅 앱이고, 이미 라이트모드 화면이 어느 정도 만들어져 있었으니 다크모드를 붙여 보기에 딱 좋은 상황이었다.

---

### Variable 세팅부터

먼저 컬러 Variable 컬렉션을 만들고 Light / Dark 두 가지 Mode를 추가했다. Variable 이름을 정하는 것부터 고민이 필요했다. 색상 코드 그대로 쓰는 게 아니라 역할 기반 이름을 써야 했기 때문이다.



정리한 Variable 목록은 이렇다.
![[image (6).png]]

- `primary` — 브랜드 핑크. Light/Dark 모두 `#F17088`로 동일
- `bg` — 배경색. Light: `#FFFFFF` / Dark: `#2A2A2A`
- `set-bg` — 카드·섹션 배경. Light: `#FAFAFA` / Dark: `#474747`
- `text` — 기본 텍스트. Light: `#2F2F2F` / Dark: `#FEFEFE`
- `text-반전` — 반전 텍스트. Light: `#FEFEFE` / Dark: `#2F2F2F`
- `icon` — 기본 아이콘. Light: `#000000` / Dark: `#FFFFFF`
- `icon-bg-chat` — 채팅 아이콘 배경. Light/Dark 반전
- `icon-bg` — 일반 아이콘 배경. Light/Dark 반전
- `header-gray` — 헤더 보조 텍스트. Light: `#9B9B9B` / Dark: `#D6D6D6`
- `grass-bg` — 잔디 배경. Light: `#F2ECE9` / Dark: `#C7B1B1`

`text`와 `text-반전`을 따로 만든 이유가 있었다. 버튼 안에 들어가는 텍스트처럼 배경이 어두울 때 흰색이 되어야 하는 텍스트와, 일반 배경 위의 텍스트가 서로 반대 방향으로 움직이기 때문이다. 이걸 하나로 합치면 다크모드에서 깨지는 경우가 생긴다.

### 네 개 화면에 적용하고 나서

로그인, 연결, 메인, 채팅 네 가지 화면에 Variable을 적용하고 Mode를 전환해 보았다.

**로그인 화면**
![[image (4).png]]

배경이 흰색에서 `#2A2A2A`로 바뀌고, 로그인 버튼은 라이트에서 검정이었다가 다크에서 흰색으로 반전되었다. 입력창도 `set-bg` Variable 덕분에 자연스럽게 어두운 회색으로 바뀌었다. `text-반전`을 제대로 써놓은 덕분에 버튼 안 텍스트가 모드에 따라 알아서 바뀌었다.

**연결 화면**
![[image (3).png]]

"내 코드" 카드 배경이 라이트에서는 연한 핑크, 다크에서는 진한 핑크로 바뀌었다. `grass-bg` Variable을 카드 배경에 써서 자연스럽게 전환되었다. "상대방 코드 입력" 영역도 `set-bg`가 붙어 있어서 모드에 따라 함께 바뀌었다.

**메인 화면**
![[image (2).png]]

가장 복잡한 화면이었다. 커플 프로필, 대화 잔디 그래프, 하단 탭바가 모두 들어가 있었다. 잔디 그래프는 `grass-bg`와 `grass-gray`를 쓰고 있었는데, 다크모드에서 배경이 `#C7B1B1`로 바뀌면서 잔디의 대비가 자연스럽게 살아났다. 탭바 아이콘도 `icon` Variable이 붙어 있어서 자동으로 반전되었다. 

**채팅 화면**
![[image (5).png]]

채팅 화면은 `icon-bg-chat`과 `icon-chat`을 따로 만들어 둔 게 빛을 발했다. 라이트에서 검정 말풍선 + 흰색 아이콘이었던 것이 다크에서 흰색 말풍선 + 검정 아이콘으로 반전되면서 채팅 인터페이스 특유의 대비감이 유지되었다.

### 만들면서 발견한 것

Variable 이름을 처음에 잘못 짓거나, 색상 코드를 직접 쓴 요소가 하나라도 있으면 모드 전환할 때 그 요소만 혼자 남는다. 실제로 몇 군데에서 이런 일이 생겼고, 찾아서 Variable로 교체하는 작업이 필요했다.

그리고 `primary` 컬러처럼 라이트/다크에서 같은 값을 쓰는 경우에도 Variable로 관리해야 한다는 걸 알게 되었다. 나중에 브랜드 컬러가 바뀌면 한 곳만 수정하면 되니까.

결국 다크모드는 색상을 어둡게 바꾸는 게 아니라 모든 색상에 역할을 부여하는 것이었다.

> [!tip]
> Variable 이름은 색상이 아니라 역할로 짓는다. `dark-gray`가 아니라 `bg`, `set-bg`, `text`처럼. 그래야 모드가 바뀌어도 이름이 말이 된다.
