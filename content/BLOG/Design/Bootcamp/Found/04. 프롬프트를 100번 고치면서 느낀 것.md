오늘 과제는 미드저니로 브랜드 무드에 맞는 비주얼 에셋을 만들어보는 거였다. 단순히 이미지를 뽑는 게 아니라 레퍼런스 탐색 → 프롬프트 설계 → 이미지 생성 → 결과 선택까지의 흐름 전체를 경험하는 게 핵심이었다.

머릿속에 원하는 이미지가 있었다. 패션 에디토리얼 사진 위에 컴퓨터 비전 UI 레이어가 얹혀 있는 것, 마치 데이터가 사람을 분석하는 것처럼 보이는 느낌. 최근 패션 그래픽이나 브랜드 캠페인에서 자주 보이는 스타일이었다.

그런데 첫 결과물을 보고 멍했다.

---

### 레퍼런스 탐색부터 시작했다

이미지를 바로 생성하지 않고 Pinterest에서 원하는 스타일을 먼저 탐색했다. 패션 에디토리얼 이미지, 컴퓨터 비전 인터페이스, 시스템 UI 오버레이, 기술적 그래픽 요소가 결합된 레퍼런스들을 모았다.

![[Pasted image 20260323215905.png]]

이 과정이 생각보다 중요했다. 막연하게 "이런 느낌"이라고만 생각했던 게, 레퍼런스를 모으면서 "패션 사진 + 시스템 인터페이스 그래픽이 섞인 비주얼"로 언어화되기 시작했다.

---

### 말이 다르면 그림이 다르다

프롬프트는 ChatGPT로 먼저 문장을 다듬은 뒤 미드저니에 넣는 방식으로 진행했다. 처음엔 이렇게 입력했다.

*"fashion photo with tech UI overlay"*

결과는 패션 화보에 스마트폰 화면이 합성된 이미지였다. 내가 원한 건 그게 아닌데. "UI overlay"라고 했을 때 AI는 스마트폰 UI를 떠올린 거다. 나는 컴퓨터 비전 탐지 박스 같은 걸 원했는데.

프롬프트를 수정했다.

 ``` 
 computer vision interface detection box overlay 
 surveillance UI technical labels 
 editorial fashion photography 
 minimal graphic layout 
 ``` 
 
 훨씬 가까웠으나 인물이 없었다. 인물을 추가했다. 그 다음에는 조명이 이상했다. 이번에는 스타일이 너무 사이버펑크였다. 키워드를 추가하거나, 표현 순서를 바꾸거나, 스타일 요소를 줄이거나. 이렇게 계속 수정을 반복했다.

### 결과물 
두 가지 주제로 최종 이미지를 만들었다. 

**기계 고양이** — 투명한 구조 안에 기계 부품들이 보이는 고양이 형태의 이미지다. 
마치 기계 설계도나 X-ray 같은 느낌으로, 기계 구조와 동물 형태가 결합된 비주얼이 독특하게 나왔다. 브랜드 그래픽이나 포스터에도 활용할 수 있을 것 같았다.

### 그러니까 프롬프트는 결국 기획 능력이다

팀에서 디자인 방향을 설명할 때도, 클라이언트한테 제안할 때도, 결국 내가 원하는 걸 언어로 정확하게 전달하는 게 핵심이다. 미드저니 프롬프트를 고치는 과정이 그것과 똑같았다. 모호하게 말하면 모호한 결과가 나오고, 구체적으로 말하면 구체적인 결과가 나온다.

AI를 쓰면서 의외로 내 기획 능력을 연습하고 있었다.

> [!tip]
> 프롬프트를 고칠수록 아이디어가 선명해진다
> AI한테 지시하는 게 아니라, AI와 대화하면서 내가 원하는 게 뭔지 발견해 가는 과정에 가깝다
